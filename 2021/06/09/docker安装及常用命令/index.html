首先是安装，自己是Centos。国内执行安装docker

curl -sSL https://get.daocloud.io/docker | sh

修改为阿里云国内仓库

vim /etc/docker/daemon.json

添加

{"registry-mirrors":["https://n44s3r.mirror.aliyuncs.com"]}

重启

sudo systemctl daemon-reload

sudo systemctl restart docker

执行 docker info 查看是否成功使用阿里云镜像

运行hello-world

docker run -i -t ubuntu:15.10 /bin/bash

-i：在新容器内指定一个伪终端

-t:允许你对容器内标准输入进行交互

容器也可以后台运行

docker run -d ubuntu /bin/sh -c "while true; do echo hello world; sleep 1; done"

-d 参数表示在后台运行

docker ps 显示在运行的容器

docker pull training/webapp 下载镜像

docker image ls 显示已有镜像、

docker stop 8606 关闭容器

docker ps -a 查看所有已停止运行的容器

docker start 8606 启动停止的容器

docker exec -it 8606 /bin/bash 跟容器交互，执行以后看shell前缀

docker export 8606 > ubuntu.tar 导出容器

docker import http://example.com/ex

docker re -f 8606 删除容器

基础命令已经过了一遍，现在运行一个python的容器

docker run -d -P training/webapp python app.py

-d 命令已经说过，-P 命令表示随机映射主机接口

再用 docker ps进行查看

访问了一下能够正常显示 Hello world！

然后这边还能修改以后重新提交你自己的docker镜像

docker commit -m='add author' -a='rtb' 7f6 training/webapp:rtb

后面的rtb表示tag

-m：提交的描述信息。-a：镜像作者。7f6是容器的id ，后面的training/webapp:rtb 表示镜像坐标。其中 :rtb 表示tag

到现在为止都是单个容器中的操作，接下来讲一下容器互联

可以通过-p 或者 -P 设置主机和docker容器的连接。还有一种方式是通过docker network 链接

docker network create -d bridge test-net

-d 参数指定Docker网络类型。有bridge overlay

docker network ls 查看网络

创建完以后，运行容器

docker run -itd --name test1 --network test-net ubuntu /bin/bash

docker run -itd --name test2 --network test-net ubuntu /bin/bash

然后在test1容器内部执行ping test2 能够正常ping通
